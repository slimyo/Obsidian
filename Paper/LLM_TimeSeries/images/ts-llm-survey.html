01
Chronos: Learning the Language of Time Series
P1 · Token化 概率预测 Amazon 2024
核心方法
①
将连续值序列缩放后做均匀量化，映射到固定词汇表（4096 bins）
②
在 T5 架构上做语言模型风格的 Cross-Entropy 预训练
③
引入 TSMix 数据增强 + 高斯过程合成数据扩充训练集
④
推理时采样多条轨迹生成概率预测分布
任务 / 数据规模
零样本预测为主；训练集含 84K+ 多领域真实序列 + 大量合成数据
量化 = "时序词汇"，让现成 LLM 无缝预测时序，合成数据大幅降低对真实数据的依赖。
02
One Fits All: Power General Time Series Analysis by Pretrained LM
P2 · 冻结LLM 多任务 NeurIPS 2023
核心方法
①
完全冻结 GPT-2 的 Transformer 层，仅训练 Input Embedding + Output Head
②
Patch 分割 + 位置编码将时序片段对齐到 Token 空间
③
同一模型权重处理 5 类任务：预测/分类/填补/异常/短期预测
关键发现
冻结的注意力层保留了对时序 pattern 有效的归纳偏置；从头训练的专用模型反而更差。
预训练语言 Backbone 中隐含的时序先验知识超乎想象，冻结即可复用。
03
In-Context Time Series Predictor
P3 · ICL 零样本
核心方法
①
将时序值序列化为文本数字串，作为 Few-Shot 示例拼入 LLM Prompt
②
研究不同提示格式（精度/分隔符/数量）对预测精度的影响
③
无需任何训练，纯推理时利用 LLM 上下文学习能力
局限性
长序列数值精度损失严重；上下文窗口长度制约历史窗口大小；对精度格式极为敏感。
Prompt 工程本身可以是时序预测的切入点，但纯 ICL 受上下文长度瓶颈制约，适合快速验证假设。
04
ITFormer: Bridging Time Series and Natural Language for Multi-Modal QA
P4 · 多模态 QA
核心方法
①
专用跨模态编码器将时序特征投影到 LLM 语言空间
②
构建大规模时序-问答对数据集 ITData（含合成+真实）
③
多任务训练：预测、分类、趋势描述、事件定位等统一为 QA 形式
贡献
首个将时序分析全面 QA 化的工作，将 LLM 可解释性直接引入时序领域。
把时序分析包装成 QA 任务，天然获得 LLM 的推理和自然语言输出能力，可解释性大幅提升。
05
Lag-Llama: Towards Foundation Models for Time Series Forecasting
P1 · Token化 概率预测
核心方法
①
引入 Lag Feature：将多个历史滞后值拼接为协变量，捕捉季节性
②
使用 LLaMA Transformer 架构预测 Student-T 分布参数
③
在 27 个公开数据集上预训练，支持零样本/少样本迁移
设计亮点
Lag Feature 是统计预测传统（ARIMA）与深度学习的桥梁，显著增强对周期模式的捕捉。
经典统计特征（滞后项）+ 现代 Transformer，是"时序领域知识"与"通用架构"结合的典范。
06
LLM4TS: Two-Stage Fine-Tuning for Time-Series Forecasting
P2 · 适配层 预测 arXiv 2308
核心方法
①
阶段一：时序数据上的自监督预训练（Mask + 重建）适配 LLM
②
阶段二：特定下游任务的有监督微调
③
Layer Norm 调整 + Patch Tokenization 实现模态对齐
优势
两阶段策略兼顾通用性与任务专用性，显著优于单阶段微调。
两阶段范式（自监督预热 → 有监督精调）已成时序 LLM 微调的标准 recipe，值得借鉴。
07
MMAD: Benchmark for Multimodal LLMs in Industrial Anomaly Detection
P4 · 多模态 工业AD Benchmark
核心方法
①
构建覆盖 39 类工业产品的大规模多模态异常检测 Benchmark
②
评估 15+ 主流 MLLM 在感知/推理/定位三个维度的表现
③
提出 MMAD-Eval 框架，标准化多模态 AD 评测协议
核心发现
现有 MLLM 在工业异常检测上存在显著能力缺口，特别是细粒度缺陷定位。
Benchmark 揭示多模态 LLM 在工业场景的能力边界，是指导未来模型改进的重要参照。
08
Unified Training of Universal Time Series Forecasting Transformers (Moirai)
P5 · 统一 多频率 Salesforce 2024
核心方法
①
统一多变量 Patch Tokenization，支持任意数量变量和预测长度
②
Any-variate Attention：每条时序独立 Patch，变量间共享权重
③
LOTSA 数据集：27B Token 跨域预训练语料
④
混合分布输出头：统一建模连续值与离散计数类型
变量数/预测长度无关的统一架构，是时序 Foundation Model 走向通用化的关键设计。
09
Semantic-Enhanced Time-Series Forecasting via Large Language Models
P4 · 语义增强 预测
核心方法
①
自动从数值序列中提取语义描述（趋势/周期/统计量），构造 Text Prompt
②
LLM 文本表征与时序数值表征进行跨模态融合（Attention Gate）
③
联合训练预测头，最终输出数值预测
将时序"翻译"成自然语言描述再反哺预测——语言知识可以提升纯数值模型的泛化能力。
10
Time-LLM: Time Series Forecasting by Reprogramming Large Language Models
P2 · Reprogramming 预测 ICLR 2024
核心方法
①
Patch Reprogramming：将时序 Patch 通过交叉注意力映射到文本词嵌入空间
②
Prompt-as-Prefix：在输入前拼接含统计描述、任务说明的自然语言 Prompt
③
LLaMA/GPT-2 骨干保持完全冻结
成果
在 8 个标准 Long-term 预测 Benchmark 上接近或超越专用 SOTA 模型。
Reprogramming 是比微调更轻量的语言-时序对齐方案，同时给 LLM 提供自然语言"上下文"能显著提升性能。
11
TIME-MOE: Billion-Scale Time Series Foundation Models with Mixture of Experts
P1+P5 · MoE 预测 2.4B params
核心方法
①
稀疏 MoE Transformer：每 Token 激活部分专家，规模可达 2.4B 参数而推理成本远低于 Dense 模型
②
Time-300B：3000 亿 Token 跨域预训练数据集（含金融/能源/气象/交通等）
③
多分辨率预测：统一处理不同采样频率和预测步长
MoE 架构将"更大模型 = 更高成本"的魔咒打破，是时序大模型走向实用化的关键路线。
12
TimeOmni-1: Incentivizing Complex Reasoning with Time Series in Large Language Models
P4+P6 · 推理 多模态推理 2025
核心方法
①
构建包含复杂推理链（CoT）的时序 QA 数据集 TimeReasonBench
②
通过 GRPO 强化学习后训练激励 LLM 进行时序推理
③
多模态架构：时序编码器 + LLM + 推理头联合训练
核心贡献
首次将 RL 后训练（类 o1/DeepSeek-R1 路线）系统引入时序多模态领域。
时序推理 = 下一个爆发点？CoT + RL 的成功经验正在从 NLP 迁移到时序，值得高度关注。
13
Timer: Generative Pre-trained Transformers are Large Time Series Models
P1 · GPT风格 预测/填补/AD ICML 2024
核心方法
①
直接复用 GPT-style Decoder-only Transformer，无任何语言模态组件
②
Single Token = 单个时间步值（不同于 Patch 方法），保留最细粒度
③
UTSD 数据集：跨领域 1B+ Token 预训练；支持三类任务统一建模
验证
GPT 架构在时序上"Scaling Law"成立，参数量提升带来稳定性能增益。
为时序领域的 Scaling Law 提供了系统性证据，大力出奇迹的逻辑同样适用于时序。

综合启示与研究启发
🔤
启示一：Token化是关键桥梁
如何将连续数值"语言化"，决定了 LLM 能力能否有效迁移。Patch（局部窗口）已成主流，兼顾计算效率与局部语义；均匀量化（Chronos）最激进，把时序完全变成离散 Token；Lag Feature（Lag-Llama）则融合了经典统计先验。不同粒度各有权衡：细粒度保留信息完整性，粗粒度提升计算效率。
🧊
启示二：冻结 LLM 效果惊人
One Fits All 和 Time-LLM 证明：完全冻结预训练 LLM 权重，仅训练轻量适配层，可以在多个 Benchmark 上匹敌甚至超越专用模型。这意味着语言预训练隐含的注意力模式对时序有效，且参数效率极高。对于资源受限的工程实践，这是性价比最高的起点。
📈
启示三：Scaling Law 同样适用
Timer 和 TIME-MOE 均系统验证了时序领域的 Scaling Law：更大模型、更多数据带来稳定的性能提升。TIME-MOE 引入稀疏 MoE 架构，将参数量推至 2.4B 的同时保持合理推理成本，为时序大模型的工程化提供了新思路：Dense 不是唯一选项，MoE 才是 Billion-scale 的正确打开方式。
🌐
启示四：数据比架构更重要
Moirai（LOTSA 27B Token）、TIME-MOE（Time-300B）、Chronos（合成增强）均将数据规模列为核心贡献。高质量、跨域、大规模预训练数据集是时序基础模型的真正壁垒。合成数据（GP、Kernel 混合）已被证明可有效填补真实数据不足——这对工程实践意义重大：缺数据时，生成数据是可行路线。
💬
启示五：语义信息是被低估的先验
Semantic-TS、Time-LLM、ITFormer 均从不同角度验证：将时序数据的语义背景（数据集描述、统计摘要、趋势词汇）注入 LLM，可以显著提升预测精度和泛化能力。纯数值建模忽略了 LLM 最擅长的"世界知识"维度，而自然语言是激活这些知识的天然接口。
🤔
启示六：推理能力是下一个前沿
TimeOmni-1 将 CoT + RL（GRPO） 引入时序多模态领域，类比 NLP 中的 o1/DeepSeek-R1 路线。时序推理不只是"预测下一个值"，还涉及根因分析、异常解释、跨变量因果推断等复杂任务。让 LLM 对时序"真正思考"而非模式匹配，可能带来质变——这是 2025 年最值得关注的新方向。
🏭
启示七：落地场景推动范式演变
MMAD 揭示：当前 MLLM 在工业异常检测等高精度、细粒度场景下仍存在显著能力缺口，特别是空间定位和细粒度分类。这说明通用大模型在垂直领域落地还需专用数据和评测体系支撑。Benchmark 建设本身就是推动领域进步的重要贡献。
🔧
实践建议：如何选择方案
零样本/快速验证：Chronos 或 Moirai，开箱即用。
资源有限、多任务：One Fits All（冻结 GPT-2），极低训练成本。
需要可解释输出：Time-LLM + 语义 Prompt，或 ITFormer QA 框架。
追求 SOTA 规模：TIME-MOE（MoE + 300B 数据）。
工业/垂直场景：参考 MMAD 评测体系设计领域专用 Benchmark 再选型。
研究演进脉络
2022 — 萌芽期
GPT for Time Series 的初探
研究者开始尝试将 GPT/BERT 应用于时序，主要停留在直接微调阶段，缺乏系统化方法论。背景：NLP 领域 GPT-3 已证明大规模预训练的威力。
2023 Q1–Q2 — 奠基期
One Fits All · LLM4TS
One Fits All（NeurIPS 2023）首次系统证明冻结 LLM 可多任务处理时序，打破"必须从头训练"的认知定式。LLM4TS 提出两阶段微调范式，成为后续工程实践的参考模板。
2023 Q3–Q4 — 爆发期
Time-LLM · Lag-Llama · ICTS
Time-LLM（ICLR 2024）提出 Reprogramming，将时序 Patch 映射到词汇空间，成为最广泛引用的适配方法之一。Lag-Llama 融合统计滞后特征，探索概率基础模型路线。ICTS 验证 In-Context 学习的上限与瓶颈。
2024 Q1–Q2 — 规模化期
Chronos · Timer · Moirai
Chronos（Amazon）通过量化将时序完全语言化，合成数据大规模扩充。Timer（ICML 2024）验证时序 Scaling Law，GPT 架构可直接移植。Moirai（Salesforce）以 27B Token 数据 + Any-variate 架构，定义了通用时序基础模型的技术标准。
2024 Q3–Q4 — 多模态期
Semantic-TS · ITFormer · MMAD · TIME-MOE
研究重心向跨模态融合迁移：语义增强、QA 框架、工业 Benchmark 相继提出。TIME-MOE 将规模推至 2.4B 参数，用 MoE 解决效率问题，时序大模型走向工程可行。
2025 — 推理期（新前沿）
TimeOmni-1
TimeOmni-1 将 CoT + RL 后训练（GRPO）引入时序，类比 NLP 中 o1 路线的时序版本。标志着时序 LLM 从"模式拟合"迈向"真正推理"，开启新一轮范式演变。
