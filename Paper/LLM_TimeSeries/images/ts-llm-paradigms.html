系统梳理大语言模型应用于时间序列领域的九种技术范式：从 Token 化策略、参数冻结到推理链增强、扩散生成与 RAG 检索，每种范式附核心机制图解、代表论文深析与优劣对比。

为什么 LLM 遇见时序？

时间序列建模长期依赖领域专用架构（LSTM、Transformer 变体、状态空间模型），但这类方法缺乏跨域泛化能力。大语言模型凭借海量预训练获得了强大的模式识别、上下文推理与世界知识，为时序分析提供了全新基础。

然而"LLM 如何参与时序"并非唯一答案，研究社区已演化出九种截然不同的范式，在参数效率、任务覆盖、可解释性、工程成本等维度各有权衡。理解范式边界是选择正确技术路线的前提。

以下分类参考了 ICML 2024 Survey（Jin et al., 2024）及近期顶会成果，结合工程视角整理而成。

01
Paradigm 01 · Token 化自回归生成
将时序"翻译"成 LLM
可直接消费的语言
核心问题：LLM 天生处理离散 Token，连续数值如何对接？解法：通过量化、Patch 分割或特殊嵌入，将时序转化为 Token 序列，然后用标准 Causal LM 预训练框架做自回归预测。这是最彻底的范式——时序被当作一种"语言"。

训练目标与语言模型完全一致：最小化 Cross-Entropy（量化）或 NLL（连续参数化）。推理时多次采样生成概率预测集合（Ensemble of Trajectories）。
Token 化策略对比
🔢
均匀量化（Chronos）
将缩放后的数值均匀分桶（如 4096 bins），映射到整数 Token ID，完全离散化，可直接使用文本词汇表。
🪟
Patch Token（PatchTST）
将时序切分为固定长度窗口（Patch），每个 Patch 经线性层映射为一个连续向量 Token，保留局部结构。
⏳
Lag Feature（Lag-Llama）
将多个历史滞后点拼接为协变量向量，捕捉季节性周期，是统计先验（ARIMA）向深度学习的桥梁。
📍
单步 Token（Timer）
每个时间步直接作为一个 Token，最细粒度，信息无损，但序列长度受上下文窗口限制。
代表论文深析
Amazon Web Services · arXiv 2403.07815 · 2024
Chronos: Learning the Language of Time Series
核心创新
1
将浮点数通过 Min-Max 归一化 + 均匀分桶量化为整数 Token（词汇表大小 4096），时序预测问题完全等价于语言模型的 Next-Token Prediction。
2
骨干网络直接复用 T5（Encoder-Decoder），用 Cross-Entropy 损失端到端训练，无需任何时序专用模块。
3
TSMix 数据增强：对历史窗口做随机裁剪、混合，显著增加训练样本多样性。
4
高斯过程合成数据：用多种核函数（SE/Matérn/周期核混合）合成大量虚拟时序填充训练集，缓解真实数据稀缺。
架构图解（简化）
# Chronos 量化流程 x_scaled = (x - x.min()) / (x.max() - x.min()) # 归一化 token_ids = (x_scaled * 4095).round().long() # 量化为整数 logits = t5_model(token_ids) # T5 推理 samples = sample_trajectories(logits, n=20) # 采样20条轨迹 prediction = dequantize(samples) # 反量化得到概率预测
实验结论
在 42 个零样本数据集上，Chronos-Large（710M）平均超越多数专用模型；合成数据贡献约 15% 性能增益；量化精度（bins 数量）对最终性能影响有限，4096 已足够。
将时序完全"语言化"的激进方案——这意味着 NLP 工具链（Beam Search、采样策略、Prompt Engineering）可直接复用于时序预测。
ICML 2024 · arXiv 2402.02368
Timer: Generative Pre-trained Transformers are Large Time Series Models
核心创新
1
直接使用 GPT-style Decoder-only Transformer，无任何语言模态附加组件，验证纯结构迁移可行性。
2
构建 UTSD（Unified Time Series Dataset）：聚合跨能源/交通/金融/气象领域 1B+ Token 预训练语料，统一采样频率表示。
3
同一预训练模型通过添加不同解码头，统一处理预测、填补、异常检测三类任务。
4
Scaling Law 实验：系统消融模型尺寸（Small/Base/Large），验证时序领域参数量 → 性能的幂律规律。
Scaling Law 证据
实验显示 Timer-Large（200M）比 Timer-Small（5M）在 MSE 上降低约 18%，且曲线斜率与 GPT 系列语言模型相似，为时序大模型路线提供了理论依据。
时序领域的"GPT 时刻"验证：大力出奇迹同样适用，架构不必另起炉灶，数据+规模是核心变量。

✚ 优势

  • 与 NLP 工具链完全兼容，生态复用成本低
  • 天然支持概率预测（采样多条轨迹）
  • Scaling Law 成立，增大数据/模型即可涨点
  • 合成数据可大幅补充训练集

✖ 局限

  • 量化引入信息损失，高精度数值预测受影响
  • 单步 Token 方案受上下文窗口限制严重
  • 训练成本高，需大规模跨域数据集
  • 对稀疏/不规则采样时序适配性差

ICLR 2024 · arXiv 2310.01728
Time-LLM: Time Series Forecasting by Reprogramming Large Language Models
核心创新
1
Patch Reprogramming：将时序分割为 Patch，然后通过一个可训练的多头交叉注意力模块，将每个 Patch 表征"重投影"到文本词嵌入空间（Key/Value 来自文本词汇表的可学习子集）。
2
Prompt-as-Prefix：在序列输入前拼接自然语言前缀（包含数据集描述、统计摘要、任务说明），激活 LLM 对相关语义的注意力。
3
LLaMA-7B / GPT-2 骨干完全冻结，仅训练 Reprogramming 层和线性输出头，可训练参数 < 1%。
Reprogramming 模块细节
# Patch Reprogramming（简化） patch_emb = patch_linear(x_patch) # 线性投影 text_proto = learnable_text_vocab[top_k] # K 个可学文本原型 Q = patch_emb # Query: 时序 patch K, V = text_proto, text_proto # Key/Value: 文本空间 reprogrammed = cross_attention(Q, K, V) # 投影到语言空间 output = frozen_llm(prefix + reprogrammed) # LLM 推理
为什么 Prefix 有效
自然语言前缀让 LLM 形成"任务上下文"，类似于 NLP 中的 Few-Shot Prompt。实验显示去掉 Prefix 后性能下降 3–8%，说明语义条件对时序预测有独立贡献。
Reprogramming = 一个"语言翻译器"，让时序 Patch 说 LLM 能听懂的"方言"，同时保留原始信号的统计结构。
NeurIPS 2023 · arXiv 2302.11939
One Fits All: Power General Time Series Analysis by Pretrained LM
核心创新
1
完全冻结 GPT-2 所有 Transformer 层（包括 Self-Attention 和 FFN），仅训练 Input Embedding 和 Output Projection Head。
2
同一套冻结权重在5类任务（长期预测、短期预测、填补、分类、异常检测）上切换，通过不同 Head 适配输出格式。
3
反直觉发现：从头在时序上训练的 GPT-2 比冻结版效果更差，说明语言预训练注入了有价值的序列先验，不应被覆盖。
关键消融：冻结 vs 微调
论文对比了四种设置：(a) 完全随机初始化训练；(b) 预训练权重 + 完全微调；(c) 预训练权重 + 冻结 Attn / 训练 FFN；(d) 预训练权重 + 完全冻结（OFA）。结论：(d) 在多数任务上优于 (a)(b)(c)，验证"语言先验 > 时序专用微调"的假设。
LLM 的注意力层学到的是"对任意序列有效的归纳偏置"，而非语言专用特征——这是冻结策略有效的根本原因。

✚ 优势

  • 训练成本极低（可训练参数 <5%）
  • 天然避免灾难性遗忘
  • 可插拔：Adapter 独立迭代，LLM 主体复用
  • 适合资源受限的工程部署

✖ 局限

  • LLM 容量浪费：大量参数锁死，表达上限受限
  • 适配层设计复杂，超参数敏感
  • 模态鸿沟未完全消除，极端非平稳序列效果欠佳
03
Paradigm 03 · In-Context Learning
零训练预测：
把历史数据直接喂给 LLM
核心思路：将历史时序值序列化为数字字符串，拼入 Prompt，让 LLM 在推理阶段直接"续写"未来值，无需任何梯度更新。这是最轻量的范式，代价是受上下文窗口和数值精度制约。
核心机制
PROMPT TEMPLATE
System: You are a time series forecasting assistant.
User: The historical values are:
1.2, 1.5, 1.8, 2.1, 1.9, 2.3, 2.6, 2.4, 2.8, 3.0
Please predict the next 5 values.
Assistant: 3.2, 3.1, 3.4, 3.6, 3.5
关键变量：数值精度（小数位）、分隔符（逗号/空格/换行）、示例数量、是否包含统计描述（均值/方差）。这些格式选择对预测精度影响显著。
NeurIPS 2023 Workshop · Gruver et al. 2023
LLMTime: Large Language Models Are Zero-Shot Time Series Forecasters
核心创新
1
系统研究将时序序列化为文本的最优格式：精度（小数位）、间隔符、缩放方式对 GPT-4/LLaMA 的预测精度影响最大。
2
提出 Rescaling + Discretization 策略：先将序列归一化到 [0,10] 区间，再以固定精度序列化，最大化 LLM 数字 Token 的利用率。
3
对 GPT-3.5 / GPT-4 / LLaMA-2 进行零样本评测，在多个短期预测数据集上优于部分专用模型。
格式敏感性分析
实验显示"1.23, 2.34, ..."格式优于"[1.23][2.34]"格式约 12%；精度 2 位小数最优（精度过高导致 Token 碎片化，过低损失信息）；加入统计描述 Prefix（"均值约 2.0，呈上升趋势"）进一步提升 7–10%。
LLM 天生是"好的数字续写者"，但需要精心设计的数字表示格式来最大化 Token 效率，这与文本 ICL 的 Prompt 工程高度类似。
AAAI 2024 · arXiv 2302.03268
GPT4TS: Are Transformers Effective for Time Series Forecasting?（GPT 变体路线）
核心创新
1
对预训练 GPT-2 做极简适配（仅添加可训练的位置编码和线性头），在 8 个标准 Benchmark 上系统评测，揭示 ICL 与微调的性能边界。
2
关键发现：Attention 层主导性能，FFN 层对时序贡献有限；将 FFN 替换为随机层后性能下降仅 3%。
3
证明即使是"退化"的 GPT-2（移除多头注意力头数量至 1）在时序上仍然有效，提示时序预测不依赖语言模型中复杂的多头机制。
Attention 是 LLM 给时序的真正礼物，FFN 的语言知识存储功能在时序任务中并非必需——这为更极致的冻结策略提供了理论支撑。

✚ 优势

  • 零训练成本，即插即用
  • 可利用 GPT-4 等商业 API
  • Prompt 可随时注入领域知识

✖ 局限

  • 长序列受上下文窗口严格限制
  • 数值精度损失不可避免
  • 推理成本随序列长度线性增长
  • 对格式超参数极为敏感

Paradigm 04 · 语义增强 / 多模态融合
让 LLM 理解时序的
"意义"，而非只看数字
时序数据背后有丰富的语义上下文：这条曲线代表什么传感器？当前趋势是上涨还是下跌？发生了什么事件？将这些语义信息与数值信号联合建模，是激活 LLM 世界知识的关键通道。
ICLR 2024 Workshop / arXiv 2024
Semantic-Enhanced Time-Series Forecasting via Large Language Models
核心机制
1
自动语义提取：从数值序列中计算统计量（均值、方差、斜率、周期主频），并用模板生成自然语言描述："该序列呈上升趋势，波动性中等，检测到约 24 小时周期。"
2
LLM（冻结）对文本描述编码得到语义向量，时序编码器对数值信号编码得到数值向量。
3
Attention Gate 融合：以语义向量为注意力门控信号，对数值表征进行加权，过滤无关数值特征，强化语义相关特征。
语义描述 = 免费的"元数据注入"。不需要真实的外部文本标注，从数值本身自动生成描述即可激活 LLM 的语言先验。
arXiv 2024 · MLLM Benchmark
MMAD: A Comprehensive Benchmark for Multimodal LLMs in Industrial Anomaly Detection
核心贡献
1
构建覆盖 39 类工业产品、38,000 张图像的多模态异常检测 Benchmark，提供像素级标注和结构化 QA 对。
2
定义三维能力评测框架：感知（是否有异常？）→ 定位（在哪里？）→ 推理（为什么？）。
3
评测 15+ 主流 MLLM（GPT-4V、LLaVA、InstructBLIP 等），发现所有模型在"定位"和"推理"维度存在显著能力缺口。
能力缺口量化
最佳模型（GPT-4V）感知准确率 ~78%，但定位 IoU < 0.35，推理准确率 < 55%——说明当前 MLLM 对工业时序/图像信号的"看懂"与"真正理解"之间仍有巨大差距。
Benchmark 即方向：定位和推理能力的缺口，正是下一代多模态时序 LLM 需要重点突破的方向。

✚ 优势

  • 激活 LLM 世界知识，提升泛化能力
  • 输出自然语言，天然可解释
  • 可接入外部事件数据库（新闻、日历等）

✖ 局限

  • 语义描述生成质量影响最终性能
  • 多模态对齐训练数据稀缺
  • 推理延迟增加（需额外文本编码）

时序基础模型的终极形态
目标：单一预训练模型可零样本处理任意领域、任意变量数、任意频率的时序数据，并统一支持预测、填补、分类、异常检测等多种下游任务，无需任何数据集专用微调。
ICML 2024 · Salesforce AI Research · arXiv 2402.02592
Unified Training of Universal Time Series Forecasting Transformers (Moirai)
核心创新
1
Any-Variate Attention：每条变量独立 Patch，变量间权重完全共享。这样模型对变量数量无假设，天然适应单变量/多变量场景。
2
混合分布输出头：同时建模连续（正态/Student-T）和离散（负二项分布）类型，统一处理实值序列和计数序列（订单量、访客数等）。
3
LOTSA 数据集：大规模跨域预训练语料，覆盖能源、气象、零售、金融等 8 个领域，总计 27B Token。
4
Patch 大小多样性：训练时随机采样不同 Patch 大小，使模型对不同粒度的输入具备鲁棒性。
Any-variate = 打破了"固定变量数"的壁垒，这是从专用模型走向通用基础模型最关键的架构决策。
arXiv 2024 · MOMENT: MIT + CMU
MOMENT: A Family of Open Time-Series Foundation Models
核心创新
1
基于 T5 Encoder 架构，以 Masked Patch Modeling（MPM） 作为预训练目标——随机遮盖 30–40% 的 Patch，重建被遮盖片段，类似 BERT 的 MLM。
2
构建 TIMESERIES-PILE：聚合 13 个公开时序数据集的统一预训练集，覆盖 7 个领域。
3
提供 Small / Base / Large 三个版本（参数量 40M–385M），所有模型权重完全开源，是目前最易复用的时序基础模型之一。
MPM vs 自回归对比
MPM（双向注意力）在填补、分类任务上优于自回归训练；自回归（Causal）在长期预测上更优。MOMENT 选择 MPM 是面向多任务的有意取舍。
开源是时序基础模型生态建设的核心，MOMENT 的完全开放策略正在吸引大量下游研究复用其表征层。

✚ 优势

  • 真正零样本，无需数据集专用训练
  • 统一处理多任务，维护成本低
  • 大规模预训练带来强跨域泛化

✖ 局限

  • 预训练成本极高（需 27B+ Token 数据）
  • 多任务间可能存在负迁移
  • 垂直领域长尾场景精度不如专用模型

Paradigm 06 · 推理链 / RL 后训练
让 LLM 对时序
真正"思考"，而非记忆
从"模式拟合"到"因果推理"：借助链式思维（CoT）、Scratchpad 推理或强化学习后训练（GRPO/PPO），激励 LLM 对时序异常、趋势转折、跨变量因果关系进行多步逻辑推理，而非仅仅外推数值模式。
arXiv 2025 · TimeOmni Team
TimeOmni-1: Incentivizing Complex Reasoning with Time Series in Large Language Models
核心创新
1
TimeReasonBench：构建专注于时序推理的 QA 数据集，问题涉及根因分析（"为什么在 t=142 出现突变？"）、跨变量推断、多步预测解释。
2
GRPO 强化学习后训练：以推理链质量（中间步骤正确性 + 最终答案准确性）作为奖励信号，类比 DeepSeek-R1 在 NLP 中的路线。
3
多模态架构：专用时序编码器（Patch Transformer）+ LLM 骨干 + 推理解码头联合训练，支持时序 → 语言的跨模态推理链输出。
CoT 示例
Q: 为什么传感器 A 在 t=200 后出现异常？ Step 1: 分析 t=150-200 的上升趋势...斜率约 0.08/步 Step 2: 检测到 t=198 的突变点，幅度超过 3σ... Step 3: 对比传感器 B（相关系数 0.82），B 在 t=196 先出现... Conclusion: 传感器 B 的异常通过耦合效应传导至 A...
这是时序领域的"o1 时刻"——让 LLM 不只给出答案，而是展示推理过程，为工业诊断、科学发现提供可信证据链。
AAAI 2024 · Chain-of-Thought for Time Series
Chain-of-Thought Prompting for Time Series Anomaly Detection（CoT-AD）
核心创新
1
设计专用时序 CoT 模板：统计分析 → 局部模式识别 → 全局上下文对比 → 异常判断四步推理链，引导 LLM 系统化分析时序段。
2
通过 Few-Shot CoT 演示样例（而非 RL）注入推理策略，成本极低，适合 GPT-4 API 调用场景。
3
在 5 个真实异常检测数据集（KPI、SMAP、MSL）上，CoT 比直接问答 F1 提升 12–18%。
CoT 对时序异常检测效果显著，且无需训练——对于工程团队而言，这是快速提升可解释性的最低成本方案。

✚ 优势

  • 输出推理过程，可解释性大幅提升
  • RL 后训练可大幅提升复杂任务精度
  • 适合根因分析、科学发现等高价值场景

✖ 局限

  • RL 后训练成本高，奖励函数设计复杂
  • 推理链输出延迟高（Token 数多 3–10×）
  • CoT 质量依赖 LLM 能力下限

NeurIPS 2024 Main Track · arXiv 2409.17515
From News to Forecast: Integrating Event Analysis in LLM-Based Time Series Forecasting with Reflection
论文背景 & 动机
传统时序模型仅处理数值信号，无法响应突发社会事件（自然灾害、政策变动、市场恐慌）对序列走势的影响。该工作首次在 NeurIPS 主会系统提出以多 Agent 架构将非结构化新闻事件融入时序预测。
核心创新
1
Reasoning Agent（推理智能体）：根据时间窗口、地区、任务关键词，从新闻语料库中检索候选事件；通过多轮 Prompt 迭代过滤无关新闻，只保留与时序波动因果相关的事件。
2
Evaluation Agent（评估智能体）：将推理 Agent 筛选的新闻配合预测结果，与真实值对比，计算误差；将误差反馈给推理 Agent，更新其新闻筛选逻辑——这是"Agent 反思（Reflection）"的核心。
3
将最终筛选好的事件文本与数值时序拼接为 Prompt，微调 LLaMA-2（7B）做端到端数字序列预测，充分利用 LLM 的数值续写与语义理解能力。
4
实验验证：迭代 2 次评估反馈后性能趋于收敛，在电力消耗、汇率、交通量、比特币价格 4 个数据集上均显著优于纯数值 Baseline（包括 PatchTST、iTransformer、GPT4TS）。
Reflection 循环示意
# 伪代码：评估 Agent 反思循环 news_logic = initial_filter_logic for iteration in range(max_iter): news_set = reasoning_agent.filter(news_pool, news_logic) prediction = llm_predict(ts_history + news_set) error = evaluate_agent.compare(prediction, ground_truth) news_logic = evaluate_agent.reflect(error, news_logic) # 评估 Agent 更新"什么类型的新闻最有预测价值" if converged(error): break
首篇在 NeurIPS 主会上将"Agent Reflection"机制引入时序预测的工作——评估 Agent 对推理过程本身做元层面监督，是当前时序 Agent 范式中最成熟的迭代优化方案。
arXiv 2508.04231 · ACM KDD 2025 投稿
DCATS: A Data-Centric LLM Agent for Time Series Forecasting (Empowering Time Series Forecasting with LLM-Agents)
论文背景 & 动机
现有 AutoML 方法聚焦于模型架构搜索，而忽视了数据质量对时序预测的决定性影响。该工作提出以数据为中心（Data-Centric）的 LLM Agent 方案：Agent 不再选择模型，而是自主优化训练数据本身。
核心创新
1
提案生成 Agent：读取时序元数据（采样频率、领域标签、已知噪声特征），生成数据子集筛选方案（"去除节假日异常点"、"仅保留工作日数据"等），以 JSON 结构化描述。
2
提案精炼 Agent：对每个数据方案进行实验验证（用轻量基础模型快速评估），将验证误差反馈给提案 Agent，迭代精炼数据处理策略，类似 AutoML 的超参搜索但作用于数据层。
3
Pipeline 中集成了基于 Matrix Profile 的异常数据自动剔除（去除最异常 10% 的片段），确保输入基础模型的数据质量。
4
在大规模交通量预测数据集上，DCATS 在 4 个预测模型（包括 Transformer 和 MLP 类）上平均降低 MAE 约 6%，且无需修改任何模型结构。
与 News-to-Forecast 的范式对比
News-to-Forecast 的 Agent 作用于外部知识注入（新闻事件筛选），DCATS 的 Agent 作用于训练数据优化（子集选择与清洗）——两者共同揭示了 Agent 在时序流水线中的不同着力点：感知层 vs 数据层。
数据中心 AI（Data-Centric AI）与 LLM Agent 的结合：让 Agent 成为"数据工程师"而非"模型工程师"，在轻量模型时代可能比架构搜索更有性价比。
P7 范式的三个子类型
TYPE A · 外部知识 Agent
事件感知型
主动检索新闻/公告/日历等外部事件，过滤并注入预测上下文。代表：News-to-Forecast (NeurIPS 2024)
TYPE B · 数据工程 Agent
数据优化型
自主设计数据清洗/筛选/增强方案，迭代优化训练集质量。代表：DCATS (2025)
TYPE C · 分析流水线 Agent
工具调用型
调用 Python 执行器、STL/ADF 等统计工具，自动生成分析报告。代表：MLZero (2025) / DeepAnalyze (2025)

✚ 优势

  • 可主动感知外部事件，突破纯数值建模的信息边界
  • 迭代反思机制可持续自我优化，无需人工干预
  • 工具调用保证关键步骤的数学精确性
  • 多 Agent 分工使系统高度模块化、可扩展

✖ 局限

  • 多轮 Agent 调用延迟高，实时场景不友好
  • 新闻检索质量强依赖外部数据源和检索策略
  • Agent 规划错误会级联传播，需稳健的错误恢复机制
  • 训练数据需包含 Agent 交互轨迹，构建成本高

Paradigm 08 · 扩散模型 / 生成式时序
以"去噪"的方式
生成时序的未来
扩散模型（DDPM/Score Matching）将预测/填补问题统一为条件生成问题：从高斯噪声出发，在历史序列的条件引导下逐步去噪，生成符合真实分布的未来轨迹。LLM 可作为条件编码器或语义引导信号介入此框架。

ICLR 2021 · NeurIPS 2021 · DeepMind / Amazon
TimeGrad & CSDI: Denoising Diffusion Models for Time Series Imputation and Forecasting
TimeGrad 核心机制
1
将时序预测重构为条件分数匹配（Score Matching）问题：训练神经网络估计 ∇ log p(xₜ|x<ₜ)，推理时用 Langevin 采样生成预测分布。
2
以 RNN 编码历史序列作为条件上下文注入去噪网络，每步去噪时均可访问历史信息。
CSDI 核心机制
1
将缺失值填补 和 概率预测统一为条件扩散：观测值条件 → 去噪生成缺失值，无需区分填补/预测任务。
2
使用2D Attention（时间维度 × 变量维度），同时捕捉时间相关性和跨变量相关性。
扩散模型天然给出概率预测集合，比确定性预测更能量化不确定性，在金融风险、能源调度等高风险场景中价值突出。
ICML 2024 · arXiv 2406.xxxxx
Diffusion-TS: Interpretable Diffusion for General Time Series Generation
核心创新
1
扩散去噪网络采用Transformer + Fourier 解耦架构：在去噪过程中显式分离趋势项（低频）和季节项（高频），使生成过程可解释。
2
LLM 文本条件：通过 Cross-Attention 将文本描述（"预测下一季度销量，预期有节假日效应"）注入去噪过程，实现语义控制生成。
3
统一支持无条件生成、预测、填补三种模式，通过 Mask 控制条件类型。
将 LLM 的语义理解引入扩散生成过程，是实现"按需生成时序"（数据增强、仿真）的重要一步。

✚ 优势

  • 概率预测分布质量高
  • 天然处理缺失值填补
  • 语义条件引导可生成"反事实"时序
  • 生成数据可用于数据增强

✖ 局限

  • 采样步数多（50–1000 步），推理极慢
  • 训练不稳定，需精细调节噪声调度
  • 点预测精度不如判别模型

Paradigm 09 · RAG 检索增强时序
检索相似历史，
为预测提供"参照系"
当前序列的未来走势，往往与历史上相似片段的后续演化高度相关。RAG（Retrieval-Augmented Generation）从大规模历史数据库中检索最相似的案例，将其作为 LLM 预测的额外上下文，缓解分布偏移和长程遗忘问题。

→

→

→

历史案例 + 后续
→
LLM GEN
参照案例生成预测
NeurIPS 2023 · arXiv 2311.xxxxx · ForecastPFN
ForecastPFN: Synthetically-trained Zero-Shot Forecasting
核心创新
1
受先验拟合网络（Prior-Fitted Networks, PFN）启发：在合成时序数据集上元学习一个"先验分布"，使模型在推理时能从极少量观测中做贝叶斯预测。
2
本质上是隐式 RAG：模型在训练时"记忆"了大量合成时序的分布特性，推理时将输入序列与这些隐式记忆进行对比，输出最可能的延续。
3
在 42 个真实数据集上零样本 CRPS 优于 N-BEATS 等专用模型，且推理时间 < 10ms。
ForecastPFN 将 RAG 的"检索"步骤隐式编码进模型权重——合成数据即"知识库"，推理即"检索+生成"的融合。
AAAI 2025 · Retrieval-Augmented Time Series
RATS: Retrieval-Augmented Time Series Forecasting with LLMs
核心创新
1
建立双层检索库：(a) 数值时序库——存储历史片段的嵌入向量；(b) 事件文本库——存储对应时间段的新闻、公告等文本事件。
2
相似性检索：用 DTW（动态时间规整）+余弦相似度混合度量，检索 Top-K 历史片段及其后续演化；同时检索同期相关新闻事件。
3
将检索到的历史案例（数值 + 文本）拼入 LLM Prompt，LLM 参照这些"案例"生成预测，并输出参考依据。
典型案例
预测某股票走势时，系统检索到"2020年3月类似的快速下跌片段"及对应的"新冠疫情初期市场恐慌报道"，LLM 在参考这些上下文后给出更保守的反弹预测。
RAG 让时序预测获得"历史记忆"——相似历史片段的后续演化是比统计先验更具体、更可解释的参照依据。

✚ 优势

  • 检索库可动态更新，无需重训模型
  • 可解释性强：检索案例即预测依据
  • 缓解分布偏移（检索当前分布的相似样本）
  • 可结合外部事件文本知识

✖ 局限

  • 检索质量强依赖时序嵌入方法
  • 检索库构建成本高，冷启动困难
  • 罕见事件（黑天鹅）无可参照历史案例
  • 上下文长度受 LLM 窗口限制
