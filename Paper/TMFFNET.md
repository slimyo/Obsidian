**问题一：特征维数选择**：为什么选择这 35 维特征？是否进行过特征重要性分析或消融实验来证明这 35 维都是必要的？

**实验：** 使用随机森林分类器。使用基尼不纯度（Gini Impurity）的降低程度，自动为每个输入特征计算贡献得分。

**基尼重要性（Gini Importance）结果**

![[top15.png]]

- **MFCC 是绝对核心**：`MFCC_12` 排在第一位（0.101），且前 15 名中 MFCC 系数占据了 10 席。这说明对于声学信号（Acoustic Signals），基于梅尔倒谱的音色特征比单纯的能量大小更具区分度。
    
- **频域特征次之**：`F_SpecSkewness`（频谱偏度）和 `F_SpecBandwidth`（谱带宽）排名非常靠前（第 2 和 第 5）。这说明声音频率分布的**形状**（是否倾斜、是否分散）是机械故障的关键指征。
    
- **时域特征贡献较小**：所有的时域特征（T_Mean, T_RMS 等）均排在 15 名以后。尤其是`T_Kurtosis`（峭度）排在倒数第一。
    - **深度解析**：这不代表峭度没用，可能是因为在这个数据集中，故障导致的冲击被环境噪声掩盖，或者 MFCC 已经以更高级的方式提取了这些信息，导致随机森林认为时域指标的边际贡献较低。

- #### 后续：特征域消融（证明多域融合的必要性）

对比模型准确率：

1. **TMFFNet (Full)**：使用全部 35 维特征。
    
2. **TMFFNet (w/o MFCC)**：去掉 12 维 MFCC，只用时频域（验证声学核心特征的重要性）。
    
3. **TMFFNet (w/o Spectral)**：去掉 12 维频域特征。
    
4. **TMFFNet (w/o Time)**：去掉 11 维时域特征。
    

- **预期结论**：去掉 MFCC 后准确率会大幅下降；去掉时域特征准确率可能下降较小，从而证明模型性能主要由 MFCC 和频域形状特征驱动。

### 模型

- 模型结构![[TMFFNET.png]]
- 两大部分：
	- TransBlock
	- TResNetBlock
- TResNetBlock
	- CBR结构
		- **(Conv) - 卷积层**：用于提取特征。
		- **B (Batch Normalization) - 批归一化**：加速网络训练收敛速度，减少模型对初始权重的敏感度，提高模型的泛化能力。
		- **R (ReLU) - 激活函数**：引入非线性因素，让神经网络能够学习更复杂的特征。
	- BU-Guided Fusion自底而上的引导式特征融合
	- $$F_{i+1}^{'}=\mu *F_i+ ({1− \mu } )*u({F_{i+1}})$$其中加权系数（Adaptive Gate）：$$μ=σ(W∗Concat(F_i,u({F_{i+1}}\text{) ))}$$上采样函数u，对齐特征维度。
这相当于智能的特征混合器。它通过学习输入特征本身，生成一个空间自适应的权重图，来精细化地控制不同层级特征的融合比例，从而获得同时具备丰富细节和强语义的融合特征

|输出形状|特征维度解释|序列长度解释|网络层作用|
|---|---|---|---|
|**(N, 64, 13)**​|**基础特征**  <br>64个通道，表示网络从原始信号中提取的**底层、局部特征**（如边缘、简单模式）。|**保留较多细节**  <br>13个时间点，保留了较精细的**时序分辨率**，能捕捉**短时变化**。|通常是第一个残差块后，经过初步卷积和池化的结果。|
|**(N, 128, 7)**​|**中级特征**  <br>通道数翻倍，表示特征变得**更抽象、更具判别力**，可能组合了底层特征形成更复杂的模式。|**时间压缩**  <br>7个时间点，感受野增大，**开始忽略部分细节**，关注**稍长时段**的模式。|经过下采样（如步长为2的卷积或池化），**时序分辨率降低，特征表达能力增强**。|
|**(N, 256, 4)**​|**高级特征**  <br>256个通道，表示特征**高度抽象**，可能对应信号中的**高级语义**（如特定事件、状态）。|**高度压缩**  <br>仅剩4个时间点，每个点**对应原始信号中较长的一段**，具有**大感受野**。|网络深度增加，进一步聚合信息，**对噪声不敏感，但可能丢失快速变化细节**。|
|**(N, 512, 2)**​|**全局特征**  <br>512个通道，通常是**最后一层卷积的输出**，特征最为**丰富和全局化**，为最终分类/回归做准备。|**极度压缩**  <br>仅剩2个时间点，可以理解为捕捉了信号的**“前半段”和“后半段”**​ 的全局模式。|接近网络末端，准备通过全局池化（如`AdaptiveAvgPool1d`）得到每个样本的固定长度表示。|

- 数据集：

1. **CPD（离心泵数据集）**：自采集数据集。包含5种运行状态（正常+4种叶片折损故障），共66,000个样本。主要用于5-way（5类）分类任务。
2. **MGD（齿轮箱数据集）**：来自DCASE 2021挑战赛。包含2种运行状态（正常、故障）。主要用于2-way（2类）分类任务。


- 特性工程：
	- (N, 2048) ->(N, num_frames, 35)
	- 例：将 2048 长的信号分成 8 帧，每帧长 1024，帧与帧之间需要重叠 `1024 - 146 = 878`个点。在每帧计算多域特征。
	- **重叠采样，可保证时间连续性，避免信息在帧边界丢失。**
	```python
	  def mutiDomainFeaturesMaker(self, data, frame_len=1024, num_frames=8):  
    """  
    将 (N, 2048) 数据转换为 (N, num_frames, 35)  
    Args:        
    data: numpy array or torch tensor, shape [N, 2048]        
    frame_len: 每帧长度  
    num_frames: 输出帧数（固定）  
  
    Returns:        
    features: torch tensor, shape [N, num_frames, 35]    
    """
	```

- 训练过程
	- 小样本学习的 Episode 数据集类:
	```python
	class FewShotDataset(Dataset):  
    """  
    Few-shot episode dataset for N-way K-shot training.  
    Args:        
    data: Tensor of shape [num_samples, seq_len, feature_dim]        
    labels: Tensor of shape [num_samples], with class labels        
    episode_num: Number of episodes per epoch        
    N: Number of classes per episode (N-way)        
    K: Number of support samples per class (K-shot)        
    Q: Number of query samples per class    
    """
	```
	不直接提供所有数据，而是模拟小样本学习场景，每次迭代中动态构建一个任务。每个 Episode 就是一个独立的分类任务，包含：

- **N-way**：任务包含 **N 个类别**（5 类）。
- **K-shot**：每个类别提供 **K 个带标签的样本**作为参考，(**支持集**)。
- **Q-query**：每个类别提供 **Q 个无标签（在训练中实际有标签，用于计算损失）的样本**作为待分类目标，(**查询集**)。
    
**模型的目标**：在仅看到 **N × K**​ 个支持样本的情况下，正确分类 **N × Q**​ 个查询样本。
