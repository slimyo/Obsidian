Claude Code 确实提供了多种方式来简化确认流程,让它只在遇到**高风险操作**时才停下来等你确认。

### 方案一：Auto Mode（智能自动，推荐首选）
这是官方于2026年3月底推出的新模式，旨在解决权限确认打断心流的问题。

*   **原理**：它不像旧方案那样“一键跳过所有检查”，而是采用**大模型（LLM）分类器**智能判断。一个内置的AI模型会实时分析每个操作，自动批准安全操作，并在遇到批量删除文件、敏感数据泄露、恶意代码执行等高风险操作时主动拦截，必要时才让你介入。Anthropic调查发现，用户日常有高达93%的权限弹窗是不经阅读就点“同意”的，Auto Mode正是对这种行为的优化。
*   **开启方法**：
    *   **启动时指定**：`claude --permission-mode auto`。
    *   **会话中切换**：进入Claude Code后，按`Shift+Tab`（或`Alt+M`）即可在多种模式间切换。
*   **前置要求**：需要Claude Sonnet 4.6或Opus 4.6模型，以及Team、Enterprise或API计划。

###  方案二：精确保留特定操作确认（精细自动）
如果你不想完全交给AI判断，还可以进行非常精细的自定义控制。
#### 1. 使用 `acceptEdits` 模式 (日常开发推荐)
这是日常开发的“甜点”选项，在**文件编辑时自动确认，但执行Shell命令时会暂停询问**，在效率和安全间取得了很好的平衡。
*   **开启方法**：启动时用 `claude --permission-mode acceptEdits` 命令，或在会话中按 `Shift+Tab` 循环切换直到看到提示。

#### 2. 编写配置文件 (`settings.json`) (精细化控制)
这是实现你“让某些高级权限才确认”想法的最佳方式，能够做到**对特定操作自动放行，对特定操作坚决拦截**。具体配置如下：
*   **允许** (`"allow"`): 定义哪些命令可以直接运行。例如 `"Bash(npm run lint)"` 或使用通配符 `"Bash(npm run: *)"`。
*   **拒绝** (`"deny"`): 将危险命令加入黑名单。例如 `"Bash(rm -rf)"` 或 `"Bash(git push --force)"`。
*   **询问** (`"ask"`): 指定某些操作无论如何都需要你确认，覆盖`allow`中的全局设置。
*   **配置方法**：
    *   全局配置文件：`~/.claude/settings.json`。
    *   项目级配置文件：`<项目根目录>/.claude/settings.json`，可提交到Git与团队共享。
    *   配置文件有优先级：`settings.local.json` > `settings.json` > 全局配置，你可以按需覆盖。

### ⚠️ 方案三：`--dangerously-skip-permissions` (极限自动，谨慎使用)
这个命令提供最高权限，跳过所有检查。**强烈不推荐在日常开发中使用**，尤其是在生产环境或包含敏感数据的项目里。高风险意味着Claude可以**无需确认地执行任何Bash命令、修改或删除任何文件**。
*   **适用场景**：仅用于完全隔离的环境（如一次性Docker容器或CI/CD流水线）。使用时建议配合沙箱，例如创建一个临时目录作为工作区或限制网络。

### 💡 锦上添花：更多实用技巧

*   **持久化配置**：可将 `--permission-mode acceptEdits` 这样的参数写进 `settings.json` 的 `defaultMode` 字段，实现每次启动默认加载。
*   **锁定工具权限**：通过 `--allowedTools` 参数限制Claude可用的工具列表，实现更细粒度的权限控制，降低风险。
*   **利用Hooks自动化**：Hooks是监听特定事件（如工具执行）并触发自定义脚本的系统。你可以设置一个 `PermissionRequest` Hook，在后台自动批准特定操作，实现更灵活的自动化。
*   **命令速查表**：

| 功能            | 命令                                      |
| :------------ | :-------------------------------------- |
| 查看版本          | `claude --version`                      |
| 启动会话          | `claude`                                |
| 恢复上次会话        | `claude -c`                             |
| 更新Claude Code | `claude update`                         |
| 切换工作模式        | 会话中按 `Shift+Tab`                        |
| 查看帮助          | `claude --help`                         |
