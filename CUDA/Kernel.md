
# 归约

```cpp
// 归约求和
    for (int stride = blockDim.x / 2; stride > 0; stride >>= 1) {
        if (tid < stride) {
            shared[tid] += shared[tid + stride];
        }
        __syncthreads();
    }
```

### 注：
#### 为什么用 `>>=`而不是 `/=`？

1. **性能**：移位操作通常比除法快
    
2. **明确意图**：表示"减半"的位操作，强调是2的幂次变化
    
3. **结果一致**：对于正整数，`>> 1`和 `/ 2`结果相同

## 核心思想

将数组元素两两相加，每次迭代使参与计算的元素数量减半，直到只剩1个总和。

## 执行步骤（以256个线程为例）

**初始状态：**

```markdown
共享内存：[a0, a1, a2, ..., a255]  // 256个元素的乘积
```

**第1轮迭代（stride=128）：**

- 线程0-127执行：`shared[tid] += shared[tid + 128]`
    
- 结果：前128个位置存放了128个部分和
    

```markdown
[ (a0+a128), (a1+a129), ..., (a127+a255), 原a128-a255被覆盖但不再使用 ]
```

**第2轮迭代（stride=64）：**

- 线程0-63执行：`shared[tid] += shared[tid + 64]`
    
- 结果：前64个位置存放了64个部分和
    

**后续迭代：**

- stride=32 → 32个部分和
    
- stride=16 → 16个部分和
    
- stride=8 → 8个部分和
    
- stride=4 → 4个部分和
    
- stride=2 → 2个部分和
    
- stride=1 → 1个总和
    

## 时间复杂度优化

- **原代码**：串行循环O(n)，每个线程计算n次加法
    
- **归约版本**：并行树形归约O(log n)，仅需log₂(256)=8轮迭代
    

## 同步机制

每次迭代后调用`__syncthreads()`，确保：

1. 所有线程完成当前轮次的计算
    
2. 数据写入共享内存后，其他线程才能读取
    

## 最终结果

- `shared[0]`存放整个block的256个元素的点积和
    
- 由线程0通过`atomicAdd`写入全局内存
    

## 可视化示例

```markdown
初始: [1,2,3,4,5,6,7,8]  // 8个元素
stride=4: [1+5, 2+6, 3+7, 4+8, ...] → [6,8,10,12]
stride=2: [6+10, 8+12, ...] → [16,20]
stride=1: [16+20] → [36]  // 最终和
```

这种设计充分利用了GPU的并行特性，将计算负载均匀分配到所有线程，避免了串行瓶颈。
