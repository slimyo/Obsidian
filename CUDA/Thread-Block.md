- gridDim
	- cuda::ceil_div,向上取整
- blockDim
	- 32的倍数，default256
- threadIdx
- thread_index = blockDim * blockIdx + threadIdx

```cpp
// 线程和块是二维的
dim3 blockDim(16, 16);  // 块大小：16×16线程
dim3 gridDim(64, 64);   // 网格大小：64×64块

// 在kernel中：
int col = blockIdx.x * blockDim.x + threadIdx.x;  // 列索引
int row = blockIdx.y * blockDim.y + threadIdx.y;  // 行索引
```

## 基本概念

### SM（Streaming Multiprocessor）

- **硬件单元**：GPU中的基本计算单元
    
- **包含组件**：
    
    - CUDA核心（计算单元）
        
    - 共享内存/L1缓存
        
    - 寄存器文件
        
    - 常量缓存
        
    - 纹理缓存
        
    
- **功能**：执行线程块
    

### Thread Block（线程块）

- **软件概念**：线程的逻辑分组
    
- **包含**：1-1024个线程（取决于GPU架构）
    
- **组织方式**：1D、2D或3D网格
    

## 映射关系

### 1. 调度与执行

```markdown
GPU硬件
├── SM0
│   ├── 线程块A
│   ├── 线程块B
│   └── 线程块C（等待）
├── SM1
│   ├── 线程块D
│   └── 线程块E
└── SM2
    └── （空闲）
```

**关键机制**：

- GPU调度器将线程块**动态分配**给可用的SM
    
- 一个SM可以**同时执行多个线程块**
    
- 一个线程块**只能在一个SM上执行**，不会跨SM
    

### 2. 资源约束

每个SM的资源有限，决定它能容纳多少线程块：

|资源类型|限制因素|典型值|
|---|---|---|
|**线程数**​|SM最大线程数|1024-2048线程/SM|
|**共享内存**​|每SM共享内存大小|48-96KB/SM|
|**寄存器**​|每SM寄存器总数|64K-128K寄存器/SM|
|**线程块数**​|最大驻留线程块数|16-32个块/SM|

**调度公式**：

```markdown
SM可执行的线程块数 = min(
    最大驻留线程块数,
    floor(SM最大线程数 / 线程块大小),
    floor(SM共享内存大小 / 每块共享内存),
    floor(SM寄存器数 / 每块寄存器)
)
```

## 实际执行流程

### 示例：启动内核

```cpp
// 启动配置：2D网格，总共6个线程块
dim3 grid(3, 2);    // 3×2=6个线程块
dim3 block(256, 1); // 每个块256个线程
kernel<<<grid, block>>>();
```

### 硬件执行过程

```markdown
时间轴：
t0: 调度器将线程块0,1分配给SM0
    调度器将线程块2,3分配给SM1
    SM2空闲
    
t1: SM0完成线程块0 → 调度器分配线程块4
    SM1完成线程块2 → 调度器分配线程块5
    SM2仍空闲（所有块已分配）

t2: 所有线程块执行完成
```

## 共享内存的关键作用

### 共享内存的物理位置

```markdown
SM内部架构：
┌─────────────────────────────────┐
│ SM                              │
│  ┌─────────────────────────┐    │
│  │ 共享内存/L1缓存 (48-96KB) │    │
│  └─────────────────────────┘    │
│  ┌─────────────────────────┐    │
│  │ 寄存器文件 (64K-128K)    │    │
│  └─────────────────────────┘    │
│  ┌─────────────────────────┐    │
│  │ CUDA核心 (64-128个)      │    │
│  └─────────────────────────┘    │
└─────────────────────────────────┘
```

### 共享内存访问特性

1. **块内共享**：同一线程块的所有线程共享相同的共享内存空间
    
2. **块间隔离**：不同线程块的共享内存**物理上独立**
    
3. **生命周期**：线程块执行期间存在，块结束时释放
    
4. **访问速度**：比全局内存快10-100倍
