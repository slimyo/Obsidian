### 内存空间分离

GPU编程中存在两个独立的内存空间：

- **主机内存空间 (Host Memory Space)**：CPU可访问的内存
    
- **设备内存空间 (Device Memory Space)**：GPU可访问的高带宽内存
    

### Thrust容器类型

Thrust提供专门的容器来管理不同内存空间的数据：

|容器类型|内存位置|可访问性|类似标准容器|
|---|---|---|---|
|`thrust::host_vector<T>`|主机内存|CPU|`std::vector<T>`|
|`thrust::device_vector<T>`|设备内存|GPU|无直接对应|
|`thrust::universal_vector<T>`|统一内存|CPU和GPU|自动迁移|

### 关键发现

1. **性能问题**：示例中出现的性能异常（某些计算步骤耗时增加100倍）是由于`thrust::universal_vector`在CPU和GPU之间自动迁移数据导致的隐式拷贝。
    
2. **隐式拷贝机制**：
    
    - 当GPU数据被CPU函数（如`ach::store`）访问时，数据从设备内存拷贝到主机内存
        
    - 当该数据随后被GPU函数（如`ach::simulate`）访问时，数据又拷贝回设备内存
        
    - 即使数据未被修改，这种拷贝仍会发生
        
    
3. **内存访问规则**：
    
    - 设备内存空间只能从设备执行空间访问
        
    - 主机内存空间只能从主机执行空间访问
        
    - Thrust的数据移动算法（如`thrust::copy`）可以在不同内存空间之间拷贝数据
        
    

### 最佳实践建议

- 使用明确的内存空间容器（`host_vector`/`device_vector`）而不是统一内存，以获得更好的性能控制和可预测性
    
- 在代码中明确标注内存空间，提高代码可读性和维护性
    
- 避免不必要的内存空间之间的数据迁移
