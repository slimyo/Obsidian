### 局部性
- 局部性原理：程序倾向于引用临近于其他最近引用过数据项的数据项，或者这个引用数据项本身。一个编写良好的计算机程序常常拥有良好的局部性。
	- 时间局部性
	- 空间局部性
- 有良好局部性的程序比局部性差的程序运行快
- 局部性原理运行计算机设计者引入**高速缓存存储器**保存最近被引用的指令，数据，提高对主存的访问速度。同理，通过主存缓存磁盘中数据。
- 注意：
	- 重复引用相同变量的程序有良好时间局部性
	- 步长为k的引用模式，步长越小，空间局部性越好。
	- 对于取指令来说，循环有良好时间和空间局部性。循环体越小，循环迭代次数越多，局部性越好

- **程序数据引用的局部性**
- 步长为1的引用模式：顺序引用模式
- 步长为k的引用模式：每隔k个元素进行访问
- ```cpp
  ----k-step
  for(int j=0;;){
	  for(int i=0;;){
		  a[i][j]
	  }
  }
  ----1-step
    for(int i=0;;){
	  for(int i=j=0;;){
		  a[i][j]
	  }
  }
  ```

- **取指令的局部性**
- 程序指令放到内存中，CPU必须取出指令

### 存储器的层次结构

![[memory.png|800]]

- 存储器层次结构中的缓存
	- 高速缓存（cache，读cash）是一个小而快速的存储设备，作为存储在更大，更慢的设备中数据对象的缓冲区域。
	- 结构的中心思想：层次结构的每一层缓存来自较低一层的数据对象
	- ![[cache.png|500]]
	- 任何时刻，第k层缓存包含第k+1层块的一个子集的副本
- 缓存命中
	- 程序需要第k+1层某数据对象d时，先在当前存储在k层的一个块中寻找d
	- 缓存命中：如何d刚好在k层中
- 缓存不命中
	- 从第k+1层取出包含d的那个块，如果k层缓存满了，则可能覆盖现在的一个块
	- 替换/驱逐块：依据替换策略
	- 放置策略：确定从k+1层的某个块放到哪里，如随机放置——不限制（对于高层，随机防止代价昂贵）
	- 不命中种类：
		- 强制不命中/冷不命中：k层为空(冷缓存)
		- 冲突不命中：缓存足够，但由于防止策略限制导致冲突
- 缓存管理：
	- 管理块、判断命中
- 高速缓存存储器
	- L1高速缓存，访问速度几乎和寄存器一样，约4时钟周期
	- L2高速缓存，约10个时钟周期
	- L3高速缓存，约50给时钟周期
