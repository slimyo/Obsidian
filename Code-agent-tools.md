
## 1,CodeGraph 

- [colbymchenry/codegraph：Claude Code、Codex、Gemini、Cursor、OpenCode、AntiGravity、Kiro和Hermes Agent的预索引代码知识图谱——更少的标记，更少的工具调用，100%本地化](https://github.com/colbymchenry/codegraph)
- 构建项目代码地图，减少编写代码时，检索项目的llm token消耗
- 使用指令：
```bash
cd your-project
codegraph init -i
```
### 工作原理：

- CodeGraph 是一个为代码库构建静态分析图并自动同步更新的工具，核心分为四个阶段：
### 1. 提取（Extract）
- 将源代码解析成 **AST**（抽象语法树）。
- 使用**语言特定的查询**从 AST 中提取两类信息：
  - **节点**：函数、类、方法等代码实体。
  - **边**：调用关系、导入关系、继承（扩展/实现）关系等。
### 2. 存储（Store）
- 所有提取出的图数据（节点+边）都存放在**本地 SQLite 数据库**中。
- 数据库路径为：`.codegraph/codegraph.db`
- 支持 **FTS5 全文搜索**，方便快速检索代码符号。
### 3. 解析（Resolve）
- 对提取到的原始引用进行**静态解析**，将符号引用链接到具体定义：
  - 函数调用 → 函数定义
  - 导入语句 → 源文件
  - 类继承关系
  - 框架特定的模式（如依赖注入、装饰器等）
### 4. 自动同步（Auto-sync）
- **MCP 服务器**通过监听操作系统的**文件事件**（如修改、新增、删除）实时感知项目变化。
- 采用**防抖策略**：等待代码变化后 **2 秒安静窗口**再触发同步，避免频繁构建。
- **增量更新**：仅对发生变化的源文件重新提取和解析。
- 整个过程**无需手动配置**，图数据会随着你编写代码自动保持最新。
> 一句话总结：CodeGraph 将代码库解析为带语义关系的图结构，存入本地 SQLite，并通过文件监控实现近实时的增量同步，让你在编码时始终拥有准确的代码依赖视图。
