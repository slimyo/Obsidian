# 强弱引用

**Weak ref**（弱引用，Weak Reference）是编程中一种特殊的**对象引用**方式，它和普通的“强引用”（strong reference）完全不同。

### 简单解释（通俗版）

- **强引用**（普通变量）：你正常写 a = SomeObject()，这个引用会“拽住”对象不让它被垃圾回收（GC）。只要还有强引用，对象就一直活着，占用内存。
- **弱引用**（weak ref）：它只是“偷偷看一眼”这个对象，不会增加对象的**引用计数**（在 Python 等语言中）。如果其他强引用都消失了，垃圾回收器可以随时把对象销毁掉，即使弱引用还存在。

结果就是：弱引用不会阻止对象被回收。当对象被回收后，弱引用会自动变成“无效”状态（通常返回 None 或抛出异常）。

### 为什么需要 Weak Ref？

主要用途是**避免内存泄漏**，特别适合以下场景：

- 实现**缓存**（cache）：存一些大对象，但不想因为缓存里引用了它，就让它永远不被回收。
- 解决**循环引用**（circular reference）：两个对象互相引用时，用弱引用断开其中一条链，垃圾回收器就能正常清理。
- 观察者模式、映射表等：需要知道对象是否存在，但不希望强行“保住”它。
