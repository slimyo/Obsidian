## Codex CLI 常用设置指南（基于官方 `-h` 输出）

### 🔑 核心概念：询问模式 + 沙箱策略

- **询问模式**（`-a, --ask-for-approval`）：控制模型生成的命令是否需要人工批准。
- **沙箱策略**（`-s, --sandbox`）：限制命令可以访问的文件系统范围（读/写）。

二者配合使用可实现从“完全手动”到“全自动”的安全级别。

---

### ⚙️ 询问模式 `-a` 可选值

| 值            | 行为（基于 CLI 描述）            |
| ------------ | ------------------------ |
| `untrusted`  | **始终**请求批准（最安全）          |
| `on-failure` | 仅在命令执行**失败**时请求批准        |
| `on-request` | **模型可主动请求**批准（默认行为）      |
| `never`      | **从不**请求批准（完全自动，但仍受沙箱限制） |

---

### 📁 沙箱策略 `-s` 可选值

| 值 | 权限范围 |
|----|----------|
| `read-only` | 只能读文件，禁止任何写入 |
| `workspace-write` | 只允许写入 `-C` 指定的工作区（及 `--add-dir` 额外目录） |
| `danger-full-access` | 允许访问整个文件系统（谨慎使用） |

---

### 🛡️ 日常推荐组合

| 场景              | 命令                                                 | 说明                                     |
| --------------- | -------------------------------------------------- | -------------------------------------- |
| **安全半自动**       | `codex -a on-request -s workspace-write`           | 模型可自行执行低风险命令，高风险时询问你。默认行为，常可省略 `-a`。   |
| **全自动（CI/CD）**  | `codex -a never -s workspace-write`                | 从不询问，自动执行所有命令，但写入范围被限制在工作区。            |
| **只读审查**        | `codex -a untrusted -s read-only`                  | 每个命令都需确认，且无法修改任何文件。                    |
| **极端危险（仅隔离环境）** | `codex --dangerously-bypass-approvals-and-sandbox` | 绕过**所有审批和沙箱**，拥有完整系统权限。仅用于一次性容器或离线虚拟机。 |

---

### 📋 常用指令速查表

| 指令                                                 | 说明                  | 注意                                   |
| -------------------------------------------------- | ------------------- | ------------------------------------ |
| `codex -a on-request -s workspace-write`           | 交互式会话，默认安全策略（可省略参数） | 这是**默认行为**，不写 `-a` 等同于 `on-request`。 |
| `codex -a never -s workspace-write`                | 全自动执行，但只能写工作区       | 适合自动化脚本，但需确保命令不会损坏工作区外文件。            |
| `codex -a untrusted -s read-only`                  | 只读模式 + 每步确认         | 非常安全，但每次命令都需人工点“同意”。                 |
| `codex --dangerously-bypass-approvals-and-sandbox` | 无限制权限（**极度危险**）     | **切勿在本地或敏感系统使用**。需自行在容器/虚拟机中运行。      |
| `codex -m gpt-4o`                                  | 指定模型为 `gpt-4o`      | 模型名取决于服务提供商。                         |
| `codex --search`                                   | 启用实时网络搜索            | 模型可使用 `web_search` 工具，需服务端支持。        |
| `codex -C /path/to/project`                        | 设置工作目录（沙箱写范围）       | 与 `-s workspace-write` 配合可限制写入路径。    |
| `codex resume --last`                              | 恢复上一次会话             | 快速继续未完成的任务。                          |
| `codex exec "命令"`                                  | 非交互式执行单次任务          | 替代 `codex [PROMPT]` 进入交互会话。          |

---

### 💡 一句话总结

**日常使用最安全的组合**：  
`codex -a on-request -s workspace-write`（或直接 `codex` 因为这是默认值）。  
**全自动任务**：`codex -a never -s workspace-write`。  
**不要碰** `--dangerously-bypass-approvals-and-sandbox` 除非你知道自己在做什么。
